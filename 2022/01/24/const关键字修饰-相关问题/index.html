const关键字修饰 相关问题

const修饰问题很麻烦，一个不错的识别方式是：const 修饰的是后面紧接着的东西

下面举 引用/指针/变量 的例子：

int const &p 和const int &p

  • int const &p 就是 int （const &p)，代表p的值不可以被修改(即不能出现p = 5这种操作);

  • const int &p 就是(const int) &p，代表const int类型的引用，p只能对const int类型引用。

int const *a和int* const a

  • int const *a，实际上可以看成是int (const *a)，这表示指针a所指向的地址可以变，但是所指向的那个值不能变。
  • int* const a，可以看成int* (const a)，我们都知道a的值其实是一个地址，这就表示a所保存的地址是不可以变的，但是这个地址对应的值是可以变的。

int const a和int const a

这俩是完全一致的，因为无论是：

  • int (const a)表示 a的值不可以变
  • 还是(const int) a 表示a的类型是const int，也表示a的值不可以变。
